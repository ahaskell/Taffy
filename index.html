Taffy by Adam Tuttle

A RESTful web service framework for ColdFusion

When it comes to ColdFusion, writing SOAP web services is easy, but consuming them can be difficult. On the other hand, writing REST web services is usually a little difficult, but consuming them is easy.

Taffy is an attempt at making writing RESTful web services in ColdFusion easy and low-friction.

Taffy is inspired largely by the FW/1 framework for MVC applications by Sean Corfield and the PowerNap framework for REST by Brian Carr. Taffy is built to be lightweight, easy to develop with, and fast. Taffy is a basically just 3 classes, one of which is optional.

First, you have the core "Api" class that the Application.cfc of your API application extends. It implements all of the logic to wire together a response for the requested URI, as well as a few helper methods for setting up the framework.

Secondly, there is a "RestApi" class, that each endpoint (cfc) extends, which implements a helper method enabling your services to return complex data types that will automatically be serialized to the requested data format (json, etc).

Lastly, there is a "GenericRepresentation" class, used to serialize complex data objects into a string representation. This is used by default, but can be overridden with a custom class to serialize to other/custom formats.

Dependencies

A CFML Engine, such as Adobe ColdFusion, Railo, or Open Blue Dragon

Install

Unzip the Taffy folder to the root of your website. Your API's Application.cfc should extend "taffy.core.api"; and each service CFC should extend "taffy.core.restapi".

Collections and Members must be implemented as separate CFCs. To map a URI to a CFC, put custom metadata on the cfcomponent tag, like so: 

<cfcomponent taffy_uri="/artist/{artistId}">

Lastly, each member/collection CFC should implement up to 4 methods, named: get, put, post, and delete. These map to the verbs allowed for that endpoint. Any un-implemented functions will be verbs that are denied for the endpoint.
